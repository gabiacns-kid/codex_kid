# Skill 학습 정리: 2026-05-15 ~ 2026-05-21

이 문서는 기존 Skill/도메인 지식 정리 이후부터 2026년 5월 21일까지의 대화와 산출물에서 반복적으로 확정된 실무 규칙을 GitHub 공유용으로 정리한 것이다.

## 1. 기본 판단 원칙

- 제안 전략은 "가능한 광고"와 "의미 있는 광고"를 구분한다.
- 예산이 작거나 전환 데이터가 부족한 경우, 매체를 넓히기보다 핵심 채널에서 판단 가능한 데이터 볼륨을 먼저 확보한다.
- 매체 제안은 클릭 수보다 CPL, CPA, ROAS, 상담 연결, 방문, 계약 등 실제 영업 단계와 연결해 설명한다.
- 광고 상품을 나열하기보다, 광고주의 업종, 예산, 전환 경로, 랜딩페이지, 추적 가능성, 운영 리소스를 기준으로 제안한다.
- 플랫폼 상품/정책/프로모션처럼 변동 가능성이 있는 내용은 공식 도움말, 공지, 상품 소개를 확인한 뒤 확정한다.

## 2. 나인블럭 미디어믹스 제안 학습

### 예산 판단

- 최소/적정/최대안은 단순 금액 차이가 아니라 데이터 확보 가능성과 운영 목적 차이로 설명한다.
- 예산이 너무 낮아 Google PMax, Demand Gen, Meta 리드폼 등 자동 최적화 상품이 작동하기 어려운 경우, 고객 요청을 그대로 반영하지 말고 현실적 한계를 명시한다.
- 견적 표기는 VAT 포함 기준으로 통일한다.
- 운영대행 수수료가 발생하는 매체는 매체비와 별도로 고객 부담 구조를 명확히 안내한다.

### 매체별 운영

- 네이버는 파워링크, 브랜드검색, 파워컨텐츠를 가맹점 모집 퍼널에 맞춰 배치한다.
- 파워링크 키워드는 대/중/소 범위와 의도/유형을 함께 고려해 그룹화한다.
- 파워컨텐츠 키워드도 검색 의도별로 나누고, 블로그와 가맹 홈페이지를 연결하는 중간 설득 지점으로 활용한다.
- Meta와 Google은 자동화 타겟만 제안하지 말고, 별도 수동 타겟 캠페인과 관심사/세그먼트 예시를 함께 제시한다.
- Google 리드폼 활용은 누락되기 쉬우므로 PMax 또는 Demand Gen 운영안에 명시한다.
- Kakao 브랜드메시지는 일부 광고계정 CBT 상품이며, 사업자의 광고성 정보 수신에 동의한 유저를 대상으로 발송할 수 있다는 조건을 참고 사항으로 둔다.

## 3. WSEKR 운영 전략 학습

### 제안 흐름

- WSEKR 제안은 회사 소개나 일반 매체 소개보다 성과 진단과 개선 구조를 먼저 제시한다.
- 핵심 흐름은 다음 순서가 자연스럽다.
  1. 현재 성과와 데이터 한계 확인
  2. 검색 의도와 캠페인/그룹 구조 진단
  3. 키워드 유형별 소재와 랜딩페이지 연결
  4. ADVoost 확장 키워드 관리
  5. 자동입찰을 통한 순위와 CPC 관리
  6. 파워컨텐츠/GFA/카카오 등 보조 지면 테스트
  7. POWER BI, HubSpot, 네이버 광고 데이터를 결합한 리포트

### 데이터와 리포트

- 네이버 광고 데이터는 노출, 클릭, 비용, CTR, CPC 등 매체 지표를 판단하는 데 사용한다.
- 전환과 매출 단계는 POWER BI 또는 HubSpot 등 WSEKR 내부 전환 데이터를 함께 봐야 한다.
- 보고서 지표 표기는 Imp, Click, CTR, CPC, Cost, Lead, Contract, CPL, CPA, ROAS 등으로 통일한다.
- 데이터가 제한적인 경우에는 "확정 진단"이 아니라 "제공 데이터 기준의 운영 방향 제안"으로 표현한다.

### 키워드와 랜딩 전략

- 키워드는 단순 개수 확장이 아니라 유형별 관리가 핵심이다.
- WSEKR은 지역, 온라인/OCC, 성인/직장인, 비즈니스, 레벨테스트, 일반 영어회화 등 검색 의도별로 분류한다.
- 각 키워드 그룹은 WSEKR 사이트 내 관련 메뉴/페이지로 맞춤 랜딩한다.
- 성과가 저조한 그룹은 소재, 랜딩페이지, 순위, 노출 시간대, 키워드 제외/직접등록 전환을 단계적으로 검토한다.
- 전환 가능성이 높은 핵심 키워드는 자동입찰로 안정적인 노출 순위를 관리한다.

### A/B 테스트

- A/B 테스트는 단순 소재 문구 테스트에 한정하지 않는다.
- 테스트 대상은 키워드, 희망 순위, 소재, 랜딩페이지, 전환 동선, 노출 시간대까지 포함한다.
- 테스트 우선순위는 다음과 같이 잡는다.
  1. 과금 비중은 높으나 전환이 저조한 키워드
  2. 클릭수는 높으나 전환이 저조한 키워드
  3. 노출 순위는 낮으나 전환율이 높은 키워드
- 테스트 기간은 기본 2주 단위로 보되, 모든 테스트를 순차 진행하는 것처럼 보이지 않게 병렬 운영 가능성을 열어둔다.
- 실험 통제 원칙은 변경 항목과 고정 항목을 구분하는 방식으로 설명한다.

## 4. Daiad Pro / 네이버 자동입찰 학습

- 네이버 검색광고가 포함된 제안서에서는 가비아CNS의 다이애드 서비스를 함께 언급한다.
- Daiad Pro는 파워링크, 쇼핑검색 상품형, 파워컨텐츠, 플레이스 자동입찰을 지원하는 것으로 정리한다.
- WSEKR에서는 파워링크가 주력 채널이므로 자동입찰의 핵심 역할을 순위 유지, CPC 통제, 고성과 키워드 노출 공백 최소화로 설명한다.
- 파워컨텐츠 자동입찰은 정보 탐색 단계의 잠재 고객을 고품질 리드로 전환하는 보조 장치로 설명한다.
- 플레이스 자동입찰은 지점 방문과 센터 활성화 관점에서 연결할 수 있다.
- 라이브 일반설정은 상시 동일 입찰, 라이브 고급설정은 요일/시간대별 세밀 조정, 레귤러 일반설정은 하루 제한적 입찰 관리, 부분설정은 기존 설정 일부 수정에 적합하다고 설명한다.
- 최대입찰가, 입찰가감액, 하향입찰은 비용 통제와 순위 유지의 균형 장치로 설명한다.

## 5. 네이버 에코시스템 운영 학습

- WSEKR은 네이버 파워링크를 중심으로 운영하되, 정보 탐색이 많은 영어 교육 업종 특성을 고려해 파워컨텐츠를 보조 지면으로 활용한다.
- 파워컨텐츠는 단순 블로그 활성화가 아니라 탐색 단계 사용자를 리드 전환 전 단계에서 설득하는 콘텐츠 경로로 설명한다.
- GFA와 커뮤니케이션애드는 WSEKR 잠재 고객이 활동할 법한 네이버 내 지면을 제한적으로 검증하는 방식으로 제안한다.
- 카카오는 현시점에서 보조 지면으로 접근하고, 브랜드메시지는 CBT 및 광고성 정보 수신 동의 조건을 확인한 뒤 테스트 집행을 검토한다.
- 신규 매체는 본예산 확대보다 제한 테스트, KPI 게이트, 중단/확대 기준을 함께 제시한다.

## 6. PPT 제작 및 QA 학습

- WSEKR/GabiaCNS PPT 수정 시 기존 글꼴, 글씨 크기, 수동 보정 위치는 사용자가 명시하지 않는 한 건드리지 않는다.
- WSEKR PPT 기준 글씨 크기는 제목 36, 카드/섹션 제목 18, 본문 15, 캡션/푸터/라벨 12로 정리한다.
- 슬라이드 하단에는 다른 장과 동일한 정리용 단어와 한 줄 캡션 구조를 유지한다.
- 콘텐츠는 좌측 상단에 몰리지 않게 한 장에 하나의 메시지가 중앙 기준으로 안정적으로 보이게 배치한다.
- Canva AI나 Gamma용 프롬프트는 추상적 지시보다 슬라이드별 구도, 텍스트, 카드 수, 배치, 금지 사항을 구체적으로 작성한다.
- PowerPoint 복구 메시지가 발생한 경우에는 사용자가 PowerPoint에서 복구 후 저장한 파일을 새 기준 파일로 삼는다.
- PPTX 수정 후에는 `presentation.xml`, `presentation.xml.rels`, `[Content_Types].xml`, `slide*.xml.rels`의 누락 관계를 점검한다.
- 새 슬라이드 추가 시 기존 배경, 폰트 크기, 하단 정리 문구 구조, 푸터 체계를 반드시 맞춘다.

## 7. 문서 작성 표현 학습

- 외부 전달용 문서에서는 내부 비판, 공식 확인용 참고 표, 불필요한 검토 메모를 제외한다.
- 기존 운영사를 비판하는 표현은 피하고, "통폐합 여지", "운영 방향 제안", "추가 검증 필요"처럼 중립적이고 실무적인 표현을 사용한다.
- "정제", "정합성"처럼 어색하거나 설명이 필요한 단어는 피하고, "선별", "개선", "맞춤도", "연결성", "운영 적합성"처럼 제안서에서 자연스러운 용어를 쓴다.
- "데이터는 근거, 제안은 운영체계"처럼 의미가 추상적인 표현은 그대로 쓰기보다, "데이터를 근거로 운영 방향을 제시한다"처럼 즉시 이해되는 문장으로 바꾼다.

## 8. 향후 Skill 반영 시 주의사항

- 이 문서는 영구 기억 자체가 아니라, Skill/도메인 지식으로 재사용할 수 있는 기준 문서다.
- Codex 세션이 자동으로 모든 프로젝트에 동기화되는 것은 아니므로, 다른 PC나 프로젝트에서 활용하려면 GitHub 문서 또는 Skill 파일을 다시 불러와야 한다.
- 플랫폼 명칭, 상품 조건, CBT 여부, 광고 정책은 변동 가능성이 있으므로 최종 제안 전 공식 문서 확인이 필요하다.
- 파일 생성 또는 PPT 수정 시 인코딩 문제와 PowerPoint 복구 메시지 이슈를 함께 점검한다.
